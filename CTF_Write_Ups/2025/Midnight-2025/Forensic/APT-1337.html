We see it is a:

daemon: ELF 64-bit LSB pie executable, x86-64, version 1 (SYSV), dynamically linked, interpreter /lib64/ld-linux-x86-64.so.2, BuildID[sha1]=dc56a9364e525e82c959b23d9bde58d603082af4, for GNU/Linux 3.2.0, not stripped

So we open it up in Ghidra and find: main() function

This function will:

  • Open the file /tmp/.dat
  • Initialize memory and some data
  • Read two environment variables : AES_KEY and AES_IV
  • Do an initial XOR of every byte with the constant 0xaa
  • Call pTqZL()
  • Finally the data is used as a command line string that is passed to the system() function

vNh3Z() function The function will:

  • Import 3 parameters
  • Converts each hex-pair to one byte It is not so important for our challenge

pTq7L() function This function will:

  • Encrypt the data using aes cbc encryption

This means that we can decrypt the content of .dat file reversing this process. What we do is

  1. Extract the files
  • daemon
python3 ../volatility3-2.11.0/vol.py -f challenge.lime linux.pagecache.InodePages --find /usr/sbin/daemon --dump
  • dat
python3 ../volatility3-2.11.0/vol.py -f challenge.lime linux.pagecache.InodePages --find /tmp/.dat --dump
  1. Search for the environment keys (I use strings but volatility can be used too)
  1. Use cyberchef:
